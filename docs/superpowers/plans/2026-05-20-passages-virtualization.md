# Passages Virtualization — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the eager full-list render in `PassagesScreen` with a TanStack Virtual-based virtualized list. Filters stay pinned at the top; the list scrolls inside its own bounded container. Visual design unchanged.

**Architecture:** Add `@tanstack/react-virtual` to the desktop app. Refactor `PassagesScreen.tsx` so the filter block, intro, and count line are static and the matching passages render inside a virtualized scroll container that only mounts the rows in view (plus overscan). Extend the layout chain (`.content` → `.screen-card` → `.passages-screen` → `.passages-list-scroll`) so the Passages screen, *and only the Passages screen*, has a bounded height with an internal scroller. Other screens keep their current page-scrolling behavior. No IPC, main-process, or core changes.

**Tech Stack:** React `^18.3.1`, TypeScript, `@tanstack/react-virtual` `^3.x`, Electron renderer. Package manager: `pnpm` workspaces. The desktop renderer has no automated UI test harness — verification is manual in the running app.

## Task 2: Add the bounded-height layout chain and virtualizer CSS

**Files:**

- Modify: `apps/desktop/src/renderer/styles.css` (around lines 345–351 for `.content`, lines 429–438 for `.screen-card`, lines 1632–1655 for the passages region)

### Why this comes before the component change

The virtualizer needs a scrollable element with a real, bounded height. Today the entire `.content` pane is the scroller and `.passages-screen` grows to content. We extend the layout chain so the Passages screen has a fixed height and the list region inside it scrolls. Scoping to a `[data-screen="Passages"]` attribute on `.content` keeps every other screen's behavior identical.

## Task 4: Refactor PassagesScreen.tsx to use the virtualizer

**Files:**

- Modify: `apps/desktop/src/renderer/screens/PassagesScreen.tsx` (full replacement)

- [ ] **Step 1: Replace the file's contents**

Replace the entire file with:

```tsx
import { useEffect, useMemo, useRef, useState } from "react";
import { useVirtualizer } from "@tanstack/react-virtual";

type Passage = {
  id: string;
  body: string;
  workId: string;
  workTitle: string;
};

type Props = {
  passages: Passage[];
  onOpenWork: (workId: string) => void;
};

export function PassagesScreen({ passages, onOpenWork }: Props): JSX.Element {
  const [query, setQuery] = useState("");
  const [workFilter, setWorkFilter] = useState("all");
  const [copiedId, setCopiedId] = useState<string | null>(null);

  const works = useMemo(
    () => Array.from(new Set(passages.map((passage) => passage.workTitle))).sort(),
    [passages]
  );
  const filtered = useMemo(() => {
    const q = query.toLowerCase().trim();
    return passages.filter((passage) => {
      const workMatches = workFilter === "all" || passage.workTitle === workFilter;
      const textMatches = !q || `${passage.workTitle} ${passage.body}`.toLowerCase().includes(q);
      return workMatches && textMatches;
    });
  }, [passages, query, workFilter]);

  const scrollRef = useRef<HTMLDivElement>(null);
  const virtualizer = useVirtualizer({
    count: filtered.length,
    getScrollElement: () => scrollRef.current,
    estimateSize: () => 180,
    overscan: 6,
    getItemKey: (index) => filtered[index].id
  });

  // When filters change, the result set may shift entirely; reset to the top.
  useEffect(() => {
    scrollRef.current?.scrollTo({ top: 0 });
  }, [query, workFilter]);

  const copyPassage = async (passage: Passage): Promise<void> => {
    try {
      await navigator.clipboard.writeText(passage.body);
      setCopiedId(passage.id);
      window.setTimeout(() => {
        setCopiedId((current) => (current === passage.id ? null : current));
      }, 1400);
    } catch {
      // Clipboard write can reject in unusual sandbox states; silently swallow rather than crash.
    }
  };

  const virtualItems = virtualizer.getVirtualItems();

  return (
    <section className="passages-screen">
      <header className="screen-intro">
        <p>Search across every ingested highlight and filter by title.</p>
      </header>
      <div className="passages-filters">
        <input
          className="library-search-input"
          placeholder="Search passages..."
          value={query}
          onChange={(event) => setQuery(event.target.value)}
        />
        <select
          className="select-input"
          value={workFilter}
          onChange={(event) => setWorkFilter(event.target.value)}
        >
          <option value="all">All works</option>
          {works.map((work) => (
            <option key={work} value={work}>
              {work}
            </option>
          ))}
        </select>
      </div>
      <p className="screen-count">
        {filtered.length} {filtered.length === 1 ? "passage" : "passages"}
      </p>
      {filtered.length === 0 ? (
        <p>No passages synced yet.</p>
      ) : (
        <div ref={scrollRef} className="passages-list-scroll">
          <div
            className="passages-list-inner"
            style={{ height: `${virtualizer.getTotalSize()}px` }}
          >
            {virtualItems.map((virtualItem) => {
              const passage = filtered[virtualItem.index];
              return (
                <div
                  key={virtualItem.key}
                  data-index={virtualItem.index}
                  ref={virtualizer.measureElement}
                  className="passages-list-row"
                  style={{ transform: `translateY(${virtualItem.start}px)` }}
                >
                  <article className="passage-card">
                    <span className="passage-card-mark" aria-hidden="true">
                      &ldquo;
                    </span>
                    <blockquote className="passage-card-body">{passage.body}</blockquote>
                    <footer className="passage-card-footer">
                      <p className="passage-card-attribution">
                        <span className="passage-card-dash" aria-hidden="true">
                          —
                        </span>
                        <span className="passage-card-title">{passage.workTitle}</span>
                      </p>
                      <div className="passage-card-actions">
                        <button
                          type="button"
                          className="passage-card-action"
                          onClick={() => onOpenWork(passage.workId)}
                          title="Open this book in Library"
                        >
                          <span className="passage-card-action-icon" aria-hidden="true">
                            ↗
                          </span>
                          Open book
                        </button>
                        <button
                          type="button"
                          className={`passage-card-action ${
                            copiedId === passage.id ? "passage-card-action-success" : ""
                          }`}
                          onClick={() => {
                            void copyPassage(passage);
                          }}
                          title="Copy quote to clipboard"
                        >
                          <span className="passage-card-action-icon" aria-hidden="true">
                            {copiedId === passage.id ? "✓" : "⎘"}
                          </span>
                          {copiedId === passage.id ? "Copied" : "Copy"}
                        </button>
                      </div>
                    </footer>
                  </article>
                </div>
              );
            })}
          </div>
        </div>
      )}
    </section>
  );
}
```

Notes on what changed vs. the original:

- The outer `<ul className="passages-list">` is now `<div className="passages-list-scroll">` with an inner `<div className="passages-list-inner">` whose height is the virtualizer's total size.
- Each row is `<div className="passages-list-row">` positioned with `transform: translateY(...)`. The card inside is an `<article className="passage-card">` instead of `<li>` (no enclosing `<ul>`). All card-inner CSS classes are unchanged, so styling is identical.
- The empty-state branch is unchanged.
- `useEffect` resets `scrollTop` to 0 on `query`/`workFilter` change.
- Existing logic — `works`, `filtered`, `copyPassage`, `copiedId` flash — is byte-for-byte preserved.

If any item fails, stop and investigate before continuing. Common failure modes:

- **Filters scroll away with the list** → the `[data-screen="Passages"]` attribute didn't land on `.content`. Recheck Task 3.
- **List doesn't scroll, full page scrolls instead** → `.passages-screen` isn't getting `min-height: 0` / `height: 100%`, or `.screen-card` isn't bounded. Recheck Task 2 Step 1 and rule ordering in Step 3.
- **Cards overlap on first paint** → `estimateSize` is too small relative to actual card heights; raise the estimate (e.g., 220). The virtualizer auto-corrects via `measureElement` after the first frame, so a brief flicker is acceptable; persistent overlap is not.
- **Other screens lost their scroll** → the `.content[data-screen="Passages"]` selector accidentally matches other screens. Confirm the attribute is set per the active screen and that the selector includes the attribute value.
